Gerard Lynn.

I build identity and authorization infrastructure, and for the last three years I have been putting AI systems into production inside a global enterprise. My working thesis: as agents take over more of the work, identity, permissions, and audit become the layer everything else depends on. Trust is a structure, not a feeling.

Twenty-five years building enterprise software, including identity infrastructure (OAuth, OIDC, SAML, Ping Identity), production LLM systems, and B2B commerce. MPS, NYU Interactive Telecommunications Program. I also make art.

Currently working on: delegated authorization for AI agents, OAuth and token exchange, auditability, and the infrastructure that makes agent deployment survivable.

MCP Harness

An MCP server platform giving AI agents structured, governed access to an enterprise codebase. Five domains: a tree-sitter code index (targeted symbol queries instead of full-file reads), Playwright browser testing, Jira integration with dry-run defaults, a documentation compiler on Claude on Bedrock that produces queryable models of routes and auth flows, and a Figma design-to-code bridge. Every write to a system of record is a proposal requiring human approval. Generalized from prototype to a configurable harness adoptable with a single project config. In active use on production accessibility remediation work. The premise: the bottleneck in enterprise AI development is not model capability but governed access, and that is an authorization problem.

Research

This grows out of the production work above: the control structures that make long-lived agent systems safe, legible, and governable.

Several of these ideas I'm developing as independent research: mechanisms that keep agent behavior stable and predictable over long-running interactions, and models of human–agent interaction built for deliberate use rather than captured attention. One project is patent pending; the rest are evolving through prototypes and technical writing rather than public repositories. Together they shape how I build production systems — observable, auditable, and designed for trust.
